# AGENTS.md — photo-swarm

Party photo-sharing over a WebRTC swarm. Read `README.md` for the full picture;
this file is the short version for agents.

## Golden rules

- **The party key `K` never reaches the server.** It lives only in the URL
  fragment (`#k=`). The server is keyed by `roomId = SHA-256(rawKey)` (opaque,
  one-way). Keep it that way.
- **Store and serve ciphertext; decrypt only for display.** Never write
  plaintext or `K` to persistent storage. All payloads/names/thumbnails go
  through `src/crypto.ts` before leaving the browser.
- **The server never inspects `signal.data`.** It relays signaling verbatim;
  authorization is implicit in knowing `roomId`. It only sets `signal.from` to
  the authenticated sender.
- **No photo bytes on the server.** It holds live sockets (ephemeral) + the
  append-only encrypted manifest (Turso) only.

## Runtimes

- **Local**: Deno. `server.ts` = `Deno.serve` + `/ws` upgrade; the rest is the
  Remix v3 fetch-router in `app/`.
- **Prod**: Cloudflare Worker + one Durable Object per room (`worker/`).

The room/session/manifest logic is shared across both runtimes
(`app/server/room.ts`, `session.ts`, `manifest-store.ts`, and
`src/protocol.ts`); only the transport wiring differs.

## Commands

```bash
deno task dev        # bundle + serve locally
deno task test       # Deno tests (crypto, manifest selectors, session flow)
deno task check      # bundle + deno check + lint + fmt --check
cd worker && npm run typecheck   # tsc against @cloudflare/workers-types
```

`deno task check` and the Deno test suite gate everything under `app/`, `src/`,
`client/`, `server.ts`. The Worker is type-checked separately (Cloudflare
globals), so run its `typecheck` after editing anything it imports.

## Conventions

- Deno-first (Web APIs), TypeScript strict, snake_case not required (mixed with
  the fetch-router `.ts` extension imports). Tests are `Deno.test` +
  `@std/assert` in `*.test.ts`.
- `public/mod.js` is generated by `deno task bundle` — never edit it, it's
  gitignored.
